## 되돌릴 수 있다는 안전망

Claude에게 큰 작업을 맡기다 보면, 몇 단계 진행한 뒤에야 "아, 이 방향이 아니었네" 하고 깨닫는 순간이 옵니다.
이때 직접 파일을 일일이 되돌리려면 번거롭고, 무엇을 바꿨는지 기억하기도 어렵습니다.

Claude Code는 이런 상황을 위해 **체크포인트(checkpoint)** 를 자동으로 만들어 둡니다.
그리고 `/rewind` 명령으로 원하는 지점의 **코드와 대화를 되돌릴 수** 있습니다.
덕분에 "되돌릴 수 있다"는 전제 아래 과감하게 작업을 시도할 수 있습니다.

---

## 체크포인트란

체크포인트는 Claude가 파일을 편집하기 **전**의 상태를 자동으로 캡처해 둔 스냅샷입니다.
별도 설정 없이 다음과 같이 동작합니다.

- 보낸 **프롬프트마다 새 체크포인트**가 생성됩니다.
- 체크포인트는 **세션을 넘어 유지**되므로, 대화를 다시 이어가도 접근할 수 있습니다.
- 세션과 함께 **약 30일 후 자동 정리**됩니다(설정으로 조정 가능).

즉, 작업 중간중간 자동으로 복원 지점이 쌓이고 있는 셈입니다.

---

## rewind 메뉴 열기

복원 메뉴를 여는 방법은 두 가지입니다.

1. 입력창에 `/rewind` 를 입력
2. 입력창이 **비어 있을 때** `Esc` 를 두 번 누르기

`Esc` 두 번이 `/rewind` 를 타이핑하는 것보다 빠릅니다.

> 입력창에 텍스트가 있을 때 더블 `Esc` 는 메뉴를 여는 대신 **입력 내용을 지웁니다.**
> 다만 지워진 텍스트는 입력 히스토리에 저장되므로, 메뉴 작업을 마친 뒤 `Up` 키로 다시 불러올 수 있습니다.

메뉴를 열면 이번 세션에서 보낸 프롬프트 목록이 나타납니다.
되돌리거나 압축할 지점을 고른 다음, 아래 동작 중 하나를 선택합니다.

---

## 복원 옵션 살펴보기

선택할 수 있는 동작은 다음과 같습니다.

| 동작 | 설명 |
| --- | --- |
| Restore code and conversation | 코드와 대화를 **모두** 해당 지점으로 되돌림 |
| Restore conversation | **대화만** 되돌리고, 현재 코드는 그대로 유지 |
| Restore code | **코드(파일 변경)만** 되돌리고, 대화는 그대로 유지 |
| Summarize from here | 선택 지점부터 **이후** 대화를 요약으로 압축 |
| Summarize up to here | 선택 지점 **이전** 대화를 요약으로 압축 |
| Never mind | 아무 변경 없이 목록으로 돌아감 |

대부분의 경우 **"Restore code and conversation"** 을 권장합니다.
대화와 파일 상태가 어긋나면 Claude의 인식과 실제 파일이 달라져 예기치 못한 동작이 생길 수 있는데,
둘을 함께 되돌리면 **Claude의 인식과 실제 파일 상태가 일치**하기 때문입니다.

대화나 `Summarize from here` 를 선택해 복원하면, 선택했던 메시지의 **원래 프롬프트가 입력창에 다시 채워집니다.**
그대로 다시 보내거나 수정해서 보낼 수 있습니다.

---

## 되돌리기(restore) vs 요약(summarize)

이름이 비슷해 헷갈리기 쉽지만, 둘은 목적이 다릅니다.

- **restore 계열**: 상태를 **되돌립니다.** 코드 변경, 대화 기록, 또는 둘 다를 실제로 undo 합니다.
- **summarize 계열**: 디스크의 파일은 **건드리지 않고**, 대화의 일부만 AI 요약으로 **압축**합니다.

요약은 컨텍스트 창 공간을 확보할 때 유용합니다.

- `Summarize from here`: 선택 메시지 **앞쪽은 원본 그대로** 두고, 선택 메시지와 그 이후를 요약으로 대체합니다.
  곁가지로 샌 토론은 버리고 초반 맥락은 자세히 남기고 싶을 때 씁니다.
- `Summarize up to here`: 선택 메시지 **앞쪽을 요약**으로 대체하고, 선택 메시지 이후는 그대로 둡니다.
  초반 설정 대화를 압축하되 최근 작업은 자세히 남기고 싶을 때 씁니다.

두 경우 모두 원본 메시지는 세션 기록에 보존되므로, 필요하면 Claude가 세부 내용을 다시 참고할 수 있습니다.
`/compact` 와 비슷하지만, 전체가 아니라 **선택 지점을 기준으로 한쪽만** 압축한다는 점이 다릅니다.

---

## 추적되지 않는 변경 (중요)

체크포인트는 **Claude의 파일 편집 도구로 이뤄진 변경만** 추적합니다.
다음 변경들은 `/rewind` 로 되돌릴 수 **없으니** 주의해야 합니다.

**1) bash 명령으로 바뀐 파일**

Claude Code가 실행한 셸 명령의 파일 변경은 추적되지 않습니다.

```bash
rm file.txt
mv old.txt new.txt
cp source.txt dest.txt
```

위와 같은 명령으로 생긴 변경은 rewind로 복구할 수 없습니다.

**2) 외부/동시 세션의 변경**

현재 세션 밖에서 직접 수정한 파일이나, 다른 세션이 동시에 바꾼 변경은
같은 파일을 건드리지 않는 한 보통 캡처되지 않습니다.

**3) 버전 관리(Git)의 대체재가 아님**

체크포인트는 어디까지나 **세션 단위의 빠른 복구**를 위한 도구입니다.

- 커밋, 브랜치, 장기 기록은 **Git 같은 버전 관리**를 계속 사용해야 합니다.
- 체크포인트는 "로컬 undo", Git은 "영구 기록"으로 생각하면 좋습니다.

## 정리

`/rewind` 는 다음 상황에서 특히 유용합니다.

| 상황 | 활용 |
| --- | --- |
| 대안 탐색 | 출발점을 잃지 않고 여러 구현 방식을 시도 |
| 실수 복구 | 버그를 만든 변경을 빠르게 되돌림 |
| 반복 실험 | 동작하던 상태로 언제든 복귀하며 변형 실험 |
| 컨텍스트 절약 | 장황한 디버깅 대화를 중간부터 요약 |

핵심은 **"되돌릴 수 있으니 과감하게 시도하라"** 입니다.
다만 bash 변경과 외부 변경은 추적되지 않으므로, **중요한 진척은 반드시 Git 커밋으로** 남겨 두는 습관이 안전합니다.
