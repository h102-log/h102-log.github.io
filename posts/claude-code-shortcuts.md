## 들어가며

Claude Code는 터미널 위에서 동작하는 도구입니다. 그래서 마우스 없이 키보드만으로 입력하고, 모드를 바꾸고, 작업을 멈추고, 세션을 정리하는 흐름이 자연스러울수록 생산성이 올라갑니다.

이 글에서는 자주 쓰는 키보드 단축키를 **입력·편집 / 특수 접두사 / 모드 전환 / 중단·되감기 / 화면 정리** 다섯 그룹으로 묶어, 각 단축키마다 간단한 사용 예시와 함께 정리합니다.

---

## 1) 입력·편집

### `Ctrl+J` — 줄바꿈

Claude Code에서는 `Enter`를 누르면 메시지가 **바로 전송**됩니다. 그래서 여러 줄짜리 프롬프트를 작성할 때는 `Ctrl+J`로 줄바꿈만 삽입합니다. 일부 터미널에서는 `Shift+Enter`도 같은 역할을 합니다.

```
> 다음 두 가지를 해줘:    ← Ctrl+J 로 줄바꿈
1. 테스트 작성
2. 문서 업데이트          ← Enter 로 전송
```

### `Ctrl+V` — 클립보드 이미지 붙여넣기

스크린샷, 에러 화면, UI 캡처를 파일로 저장하지 않고 입력창에 바로 붙여넣습니다. Claude는 멀티모달 모델이라 이미지를 그대로 분석합니다.

```
# 에러 스크린샷 복사 후
Ctrl+V
> 이 에러 화면의 원인을 찾아줘
```

### `Tab` — 자동완성

파일 경로나 슬래시 명령을 입력하다 `Tab`을 누르면 매칭되는 후보를 완성해 줍니다. 긴 경로를 일일이 칠 필요가 없습니다.

```
> @src/comp   ← Tab
> @src/components/   ← 경로 자동완성
```

### `↑` / `↓` — 명령 히스토리 탐색

`↑`로 이전 입력을 거슬러 올라가고 `↓`로 최근 입력으로 돌아옵니다. 비슷한 프롬프트를 파일명만 바꿔 반복할 때 특히 유용합니다.

```
# ↑ 로 직전 입력을 불러온 뒤 일부만 수정
> auth.py 분석해줘   →   utils.py 분석해줘   (Enter)
```

### `Ctrl+R` — 히스토리 역검색 / verbose 토글

터미널에서는 키워드로 이전 입력을 찾는 **역방향 검색(reverse-i-search)** 으로, 일부 Claude Code 버전에서는 내부 처리 과정을 보여주는 **상세 출력 토글**로 동작합니다(환경에 따라 다름).

```
Ctrl+R
(reverse-i-search)`test`: !npm test   ← 키워드로 과거 입력 검색
```

## 2) 특수 접두사 / 메뉴

입력 맨 앞에 붙이는 한 글자가 입력의 **해석 방식**을 바꿉니다.

### `/` — 슬래시 명령 메뉴

`/`를 입력하면 사용 가능한 슬래시 명령 목록이 뜹니다. 일부를 이어서 치면 실시간 필터링되고, 커스텀 명령도 함께 보입니다.

```
> /
> /clear   ← 메뉴에서 선택
```

### `@` — 파일/폴더 참조

`@`로 파일을 퍼지 검색해 선택하면 그 파일 내용이 컨텍스트에 첨부됩니다. 한 프롬프트에서 여러 번 써서 여러 파일을 붙일 수 있습니다.

```
@src/app.py 이 파일에서 에러 처리가 누락된 곳을 찾아줘
```

### `!` — bash 모드 (셸 명령 실행)

`!` 접두사를 붙이면 입력을 셸 명령으로 실행하고, 그 출력이 대화에 포함됩니다. 빌드·테스트 결과를 그대로 Claude에게 넘길 때 좋습니다.

```
!npm test
# 실패 로그가 컨텍스트에 올라옴 → "이 에러 고쳐줘"로 이어가기
```

### `#` — 메모리에 빠르게 추가

`#` 접두사를 붙이면 긴 응답 없이 그 내용을 메모리(주로 `CLAUDE.md`)에 한 줄로 기록합니다. 대화 흐름을 끊지 않고 규칙이나 선호를 저장합니다.

```
# 커밋 메시지는 항상 한국어로 작성한다
→ CLAUDE.md 에 조용히 기록됨
```

## 3) 모드 전환

### `Shift+Tab` — 입력 모드 순환

`Shift+Tab`을 누를 때마다 입력 모드가 순환합니다. 모드에 따라 Claude가 파일을 수정할 때 승인을 묻는 방식이 달라집니다.

```
기본 모드 → auto-accept 편집 → plan 모드 → (다시) 기본 모드
```

- **기본**: 변경마다 승인 요청
- **auto-accept**: 편집을 자동 수락 (속도 우선)
- **plan**: 수정 없이 계획만 수립

---

## 4) 중단·되감기

비슷해 보이지만 의미가 다른 키들입니다. 구분이 핵심입니다.

### `Esc` — 현재 작업/응답 중단

Claude가 응답을 생성하거나 도구를 실행하는 **도중**에 즉시 멈춥니다. 엉뚱한 방향으로 흘러갈 때 빠르게 끊고 바로 수정 지시를 내릴 수 있습니다.

```
Claude가 잘못된 파일을 수정하려는 중
Esc → 작업 중단, 입력창 활성화
```

### `Esc` `Esc` — 이전 메시지로 되감기·편집

`Esc`를 빠르게 두 번 누르면 **이전 사용자 메시지**를 편집창으로 불러옵니다. 수정 후 다시 보내면 그 시점 이후 대화가 새 분기로 교체됩니다.

```
Esc Esc → 직전에 보낸 프롬프트를 불러와 수정 → 재전송
```

### `Ctrl+C` — 입력 취소

작성 **중인 입력창 텍스트**를 지웁니다(작업 중단인 `Esc`와 다름). 빈 입력창에서 누르면 종료 흐름으로 이어질 수 있습니다.

```
> 이 코드를 전부 다시... (작성 중)
Ctrl+C → 입력창 비워짐
```

### `Ctrl+D` — 세션 종료 (EOF)

빈 입력창에서 누르면 EOF 신호로 세션을 정상 종료합니다. `/exit`를 치는 대신 빠르게 닫을 때 씁니다(`Ctrl+C`는 SIGINT, `Ctrl+D`는 EOF로 의미가 다름).

```
>            ← 빈 입력창
Ctrl+D → 세션 종료, 터미널로 복귀
```

---

## 5) 화면 정리

### `Ctrl+L` — 화면 지우기 (대화는 유지)

터미널 화면만 깨끗이 비웁니다. **대화 컨텍스트와 AI 기억은 그대로** 유지되므로, 출력이 쌓여 지저분할 때 세션을 새로 시작하지 않고 정리할 수 있습니다.

```
Ctrl+L → 화면만 클리어 (이전 맥락은 유지)
```

## 정리

| 단축키 | 기능 | 한 줄 요약 |
| --- | --- | --- |
| `Ctrl+J` | 줄바꿈 | Enter는 전송, 줄바꿈은 Ctrl+J |
| `Ctrl+V` | 이미지 붙여넣기 | 스크린샷을 바로 전달 |
| `Tab` | 자동완성 | 경로·명령 완성 |
| `↑` / `↓` | 히스토리 탐색 | 이전 입력 불러오기 |
| `Ctrl+R` | 역검색 / verbose | 키워드로 과거 입력 검색 |
| `/` | 슬래시 메뉴 | 명령 목록 열기 |
| `@` | 파일 참조 | 파일 내용을 컨텍스트에 첨부 |
| `!` | bash 모드 | 셸 명령 실행 |
| `#` | 메모리 추가 | CLAUDE.md에 한 줄 기록 |
| `Shift+Tab` | 모드 전환 | 기본 ↔ auto-accept ↔ plan |
| `Esc` | 작업 중단 | 실행 중인 응답 멈춤 |
| `Esc` `Esc` | 되감기·편집 | 이전 메시지 수정 후 재전송 |
| `Ctrl+C` | 입력 취소 | 입력창 텍스트 비우기 |
| `Ctrl+D` | 세션 종료 | EOF로 정상 종료 |
| `Ctrl+L` | 화면 지우기 | 대화는 유지, 화면만 클리어 |

단축키는 한 번에 외우려 하기보다, 자주 하는 동작부터 손에 익히는 것이 효율적입니다. **입력은 `@`·`!`·`Tab`, 흐름 제어는 `Esc`·`Shift+Tab`** 정도만 익숙해져도 작업 속도가 눈에 띄게 빨라집니다.
