## 한 줄 소개

아침에 알람을 끄는 순간, 오늘 비 소식을 강제로 인지하게 만들어 "우산 놓침"을 줄이는 스마트 알람 앱입니다.

---

## 프로젝트 개요

- 프로젝트명: 우산 챙겼니?
- 플랫폼: iOS / Android (React Native 기반 크로스 플랫폼)
- 기능 :
  - 사용자가 설정한 기상 시간에 알람이 울림
  - 알람 해제 시점에 날씨 정보를 함께 노출
  - 비 예보가 있으면 시각/청각적으로 강하게 경고

---

## 기획 배경과 개발 목표

### 기획 배경

아침에 알람을 끄고 출근 준비를 하다가, 날씨 앱을 확인하지 못해 비를 맞을 뻔한 적이 있었다.
알람이 울릴때 날씨정보도 알려주면 좋겠다 싶었는데 실제로 그런 기능이 있지만
이 기회에 직접 만들어서 사용해보자 생각했다.

### 개발 목표

- React Native + Expo 기반 모바일 앱 개발 사이클 전반 경험
- 외부 날씨 API 연동 및 로컬 캐싱 구조 설계
- 로컬 푸시 알람/백그라운드 동작 원리 이해
- 예외 상황(권한 거부, 네트워크 단절)에서도 앱이 깨지지 않는 방어적 처리

---

## 주요 기능 설계

### 알람 설정 및 관리

- 사용자 맞춤형 기상 시간 설정
- 요일별 반복 알람 On/Off
- 알람 목록 조회/수정/삭제

### 날씨 연동 및 우산 추천 로직

- 현재 위치(위경도) 기반 날씨 조회
- 강수 확률/날씨 상태를 분석해 우산 플래그 계산
- 예시 규칙:
  - 강수 확률 $\ge 40\%$ 또는 상태 코드가 rain/snow 계열이면 우산 추천

### 스마트 알람 해제 UX

- 알람 화면 중앙에 오늘의 날씨 + 우산 필요 여부를 즉시 표시
- 비 예보 시:
  - 시각 경고: 우산 아이콘/애니메이션 강조
  - 청각 경고: 일반 알람 대비 명확한 경고 사운드

### API TTL 캐싱

- 로컬에 최근 날씨 응답 저장
- TTL(예: 1시간) 내에는 캐시 재사용
- TTL 초과 시에만 재호출

이 방식으로 불필요한 API 호출과 배터리/네트워크 소모를 줄이고, 알람 화면 응답 속도를 안정화합니다.

## 기술 스택

- Frontend: React Native, Expo
- Navigation: Expo Router
- State Management: Zustand
- Local Storage: AsyncStorage
- API: OpenWeatherMap API 또는 기상청 API

## 개발 계획 (4주)

### 첫째 주: 기획 고도화 + 환경 세팅

- Expo 프로젝트 초기화 및 Expo Router 구조 확정
- 디자인 프레임 정리
- 기초 컴포넌트(버튼, 알람 리스트, 설정 패널) 구현

### 둘째 주: 알람 + 위치 기능 구현

- 날짜/시간 선택 UI 연동
- 로컬 알람 스케줄링 로직 구현
- 위치 권한 요청 및 좌표 획득/실패 분기 처리

### 셋째 주: 날씨 API + 알람 화면 결합

- 날씨 API 연동 및 AsyncStorage TTL 캐싱 적용
- 알람 울림 시점에 우산 추천 상태 렌더링
- 비 예보 시 경고 UI/사운드 강화

### 넷째 주: 테스트 및 최적화

- Android/iOS 실제 디바이스 테스트
- 백그라운드/권한/오프라인 예외 시나리오 검증
- 리팩토링 및 로그 정리
