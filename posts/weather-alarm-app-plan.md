## 한 줄 소개

아침에 알람을 끄는 순간, 오늘 비 소식을 강제로 인지하게 만들어 "우산 놓침"을 줄이는 스마트 알람 앱입니다.

---

## 프로젝트 개요

- 프로젝트명: 우산 챙겼니? (가칭)
- 플랫폼: iOS / Android (React Native 기반 크로스 플랫폼)
- 핵심 컨셉:
  - 사용자가 설정한 기상 시간에 알람이 울림
  - 알람 해제 시점에 날씨 정보를 함께 노출
  - 비 예보가 있으면 시각/청각적으로 강하게 경고

이 앱의 목적은 단순한 알람이 아니라, "알람 해제"라는 반복 행동에 날씨 인지를 결합해 실질적인 생활 문제를 해결하는 데 있습니다.

---

## 기획 배경과 개발 목표

### 기획 배경

아침에 무의식적으로 알람을 끄고 출근 준비를 하다가, 날씨 앱을 확인하지 못해 비를 맞을 뻔한 경험에서 출발했습니다.

문제의 핵심은 "날씨를 몰라서"가 아니라 "확인하는 행동을 놓치기 쉽다"는 점입니다.
그래서 앱의 UX 중심을 "알람 해제 시점에 날씨를 반드시 보게 만드는 흐름"에 둡니다.

### 개발 목표

- React Native + Expo 기반 모바일 앱 개발 사이클 전반 경험
- 외부 날씨 API 연동 및 로컬 캐싱 구조 설계
- 로컬 푸시 알람/백그라운드 동작 원리 이해
- 예외 상황(권한 거부, 네트워크 단절)에서도 앱이 깨지지 않는 방어적 처리

---

## 주요 기능 설계

### 알람 설정 및 관리

- 사용자 맞춤형 기상 시간 설정
- 요일별 반복 알람 On/Off
- 알람 목록 조회/수정/삭제

### 날씨 연동 및 우산 추천 로직

- 현재 위치(위경도) 기반 날씨 조회
- 강수 확률/날씨 상태를 분석해 우산 플래그 계산
- 예시 규칙:
  - 강수 확률 $\ge 40\%$ 또는 상태 코드가 rain/snow 계열이면 우산 추천

### 스마트 알람 해제 UX

- 알람 화면 중앙에 오늘의 날씨 + 우산 필요 여부를 즉시 표시
- 비 예보 시:
  - 시각 경고: 우산 아이콘/애니메이션 강조
  - 청각 경고: 일반 알람 대비 명확한 경고 사운드

### API TTL 캐싱

- 로컬에 최근 날씨 응답 저장
- TTL(예: 1시간) 내에는 캐시 재사용
- TTL 초과 시에만 재호출

이 방식으로 불필요한 API 호출과 배터리/네트워크 소모를 줄이고, 알람 화면 응답 속도를 안정화합니다.

---

## 기술 스택

- Frontend: React Native, Expo
- Navigation: Expo Router
- State Management: Zustand
- Local Storage: AsyncStorage
- API: OpenWeatherMap API 또는 기상청 API

---

## 개발 원칙 및 컨벤션

### 명명 규칙

- Logic (변수/함수): camelCase
  - 예: `fetchWeatherData`, `isUmbrellaRequired`
- Design (스타일 토큰/컴포넌트 class 계열): kebab-case
  - 예: `weather-status-card`
- Storage Key: snake_case
  - 예: `weather_cache_data`, `alarm_schedule_list`

### 코딩 가이드

- 방어적 코딩(Defensive Coding) 우선
  - API 실패, 위치 권한 거부, 오프라인 상태에서도 앱 종료 없이 fallback 처리
- 최신 문법 활용
  - ES6+ / TypeScript 기반으로 가독성과 유지보수성 확보
- 주석 기준
  - 동작 설명보다 "의도"와 "주의사항" 중심
  - 예: TTL 만료 기준, 알람 스케줄 동기화 타이밍, 실패 시 대체 흐름

## 개발 마일스톤 (4주)

### 첫째 주: 기획 고도화 + 환경 세팅

- Expo 프로젝트 초기화 및 Expo Router 구조 확정
- 디자인 와이어프레임 정리
- 기초 컴포넌트(버튼, 알람 리스트, 설정 패널) 구현

### 둘째 주: 알람 + 위치 기능 구현

- 날짜/시간 선택 UI 연동
- 로컬 알람 스케줄링 로직 구현
- 위치 권한 요청 및 좌표 획득/실패 분기 처리

### 셋째 주: 날씨 API + 알람 화면 결합

- 날씨 API 연동 및 AsyncStorage TTL 캐싱 적용
- 알람 울림 시점에 우산 추천 상태 렌더링
- 비 예보 시 경고 UI/사운드 강화

### 넷째 주: 테스트 및 최적화

- Android/iOS 실제 디바이스 테스트
- 백그라운드/권한/오프라인 예외 시나리오 검증
- 리팩토링 및 로그 정리

---

## 리스크와 대응 전략

- 위치 권한 거부:
  - 기본 지역 선택 UI 제공, 권한 재요청 동선 명확화
- 네트워크 단절:
  - 마지막 캐시 데이터 표시 + 데이터 시점 안내
- 백그라운드 제약(iOS/Android 차이):
  - OS 정책을 고려한 알람 스케줄 방식 분리
- API 할당량 이슈:
  - TTL 재사용, 재시도 횟수 제한, 오류 백오프 적용

## 마무리

"우산 챙겼니?"는 기술적으로는 알람 앱이지만, 사용자 경험 측면에서는 "실수 방지 시스템"에 가깝습니다.

핵심은 화려한 기능보다, 아침의 짧은 순간에 필요한 정보를 정확하게 전달하는 것입니다.
이 방향을 기준으로 MVP를 완성한 뒤, 사용자 피드백을 반영해 고도화해 나갈 예정입니다.
