# 개발기 #1 — 프로젝트 개요와 설계 로드맵

React에 대해 어느 정도 깊이 있게 공부했다는 생각이 들어, 이를 검증하고 실력을 다지기 위한 프로젝트를 시작해 보려고 합니다.

기존 JSP 환경과 달리, React는 상태가 변경되었을 때 변경된 부분만 업데이트하는 효율적인 렌더링 방식을 가집니다. 이러한 React만의 특징을 가장 잘 활용하고 체감할 수 있는 프로젝트가 무엇일까 고민하다가, 대용량 데이터를 빠르게 렌더링할 수 있는 DataGrid 컴포넌트를 직접 만들어보기로 했습니다.

단순한 UI 구현을 넘어, 실무에서 필수적인 데이터 내보내기 기능과 최근 트렌드인 AI 기술을 접목한 데이터 인터랙션까지 포함하는 것을 목표로 삼았습니다. 직접 만드는 과정에서 타입 시스템 설계부터 렌더링 엔진 구현, AI 기능 통합까지 단계별로 경험하며 오픈소스(npm) 배포까지 완주해 보려 합니다.

---

## 프로젝트 목표

1. **렌더링:** 가상화 기술을 적용해 대용량 데이터도 끊김 없이 처리
2. **타입 시스템 구축:** TypeScript를 기반으로 한 견고하고 유연한 API 설계
3. **데이터 EXPORT 기능:** 다양한 업무 환경을 고려한 CSV, Excel 등 데이터 추출 기능 제공
4. **AI 기능:** LLM을 활용한 자연어 데이터 필터링 및 인사이트 요약 기능 통합
5. **오픈소스 배포:** 완성된 컴포넌트를 npm 패키지로 구성하여 배포 및 문서화

---

## 개발 일정 및 단계

- **4월 말 ~ 5월 1주: 설계 및 코어 구현**
  - 기초 타입 시스템 설계 및 데이터 구조 정의
  - 가상화(Virtualization) 레이어를 포함한 핵심 렌더링 엔진 개발
- **5월 2주: 상호작용 및 데이터 활용**
  - 선택(Selection), 키보드 네비게이션 기능
  - 데이터 EXPORT 엔진 (CSV, Excel) 연동 및 유틸리티 개발
- **5월 3주: 컬럼 제어 및 필터링**
  - 컬럼 정렬(Sort), 리사이즈, 순서 변경 기능
  - 복합 필터링 시스템 구축
- **5월 4주: AI 기능 통합**
  - 자연어 쿼리를 활용한 데이터 필터링 기능 구현
  - AI 기반 데이터 요약 및 인사이트 추출 기능 실험
- **6월 1주: 완성 및 배포**
  - 전체 성능 최적화 및 유닛/통합 테스트 완료
  - 공식 문서(README) 작성 및 npm 배포

## 개발 시 원칙

- **성능 최적화에 집중:** 가상화, 메모이제이션 등 React의 최적화 기법을 적극적으로 활용하여 성능 저하 최소화
- **유연한 API 설계:** 사용자가 자신의 요구사항에 맞게 기능을 커스텀할 수 있도록 확장성 있게 설계
- **철저한 테스트:** 안정적인 라이브러리를 지향하며 유닛 테스트 및 통합 테스트 작성 생활화
- **주도적인 개발:** '바이브 코딩'은 지양하고, AI는 검색 엔진 및 논리 검증을 위한 조력자로만 활용하며 직접 고민하고 구현하기
