# [JS] Array.prototype.some() 완벽 가이드 및 실무 활용법

## 1. 정의
`some()` 메서드는 배열 안의 요소 중 **단 하나라도** 주어진 판별 함수(조건)를 통과하면 `true`를 반환합니다. 만약 조건을 만족하는 요소를 찾지 못하면 `false`를 반환합니다.

## 2. 기본 사용법
가장 기본적인 형태의 코드입니다. 배열 내에 특정 값이 존재하는지 확인할 때 직관적으로 사용할 수 있습니다.

```javascript
const arr = [1, 2, 3, 4];

// [비즈니스 로직] 배열 요소 중 1이 하나라도 존재하는지 확인합니다.
const checkOne = arr.some(val => val === 1);

console.log(checkOne); 
// 출력: true
```

## 3. 실무 적용 시 주의사항 (방어적 코딩)
실제 실무 환경에서는 API 응답 지연이나 예외 상황으로 인해 데이터가 배열 형태가 아니거나 비어있는 상태로 넘어올 수 있습니다. `some()`은 배열이 아닌 타입에서 호출 시 런타임 에러를 발생시키므로, 반드시 배열 검사를 선행하는 것이 좋습니다.

```javascript
const arr = [1, 2, 3, 4];
let checkOne = false;

// [방어적 코딩] 배열 검사 및 비어있는 배열을 대비한 예외 처리
// 데이터가 유효한 배열이고 길이가 0이 아닐 때만 some() 로직을 수행합니다.
if (Array.isArray(arr) && arr.length !== 0) {
    checkOne = arr.some(val => val === 1);
}

console.log(checkOne);
// 출력: true
```

## 4. forEach, filter와의 차이점 (조기 종료)
조건을 검사할 때 `forEach`나 `filter`를 사용할 수도 있지만, 목적에 따라 성능 차이가 발생합니다.

* **forEach의 한계:** 중간에 `return`이나 `break`로 멈출 수가 없습니다. 무조건 배열 전체를 탐색하기 때문에 데이터 크기가 커질수록 성능 낭비가 발생합니다.
* **filter의 한계:** `filter` 역시 끝까지 순회하며, 조건에 맞는 요소를 모아 **새로운 배열을 생성하여 반환**하기 때문에 불필요한 메모리와 자원 낭비가 발생합니다.
* **some의 장점 (Short-circuit):** `some()`은 조건을 만족하는 요소를 발견하는 즉시 **순회를 멈추고 `true`를 반환(조기 종료)**하므로 성능상 훨씬 유리합니다.

## 5. 일반 for 반복문과의 차이점 (가독성 vs 성능)
사실 `some`의 동작은 일반 `for` 문 안에서 `if` 조건에 맞을 때 `return true`를 해버리는 것과 완벽히 동일합니다. 

콜백 함수를 호출하지 않는 순수 `for` 문이 브라우저 엔진 레벨에서는 아주 약간 더 빠를 수 있습니다. 하지만 대규모 데이터 연산이 아닌 이상 그 성능 차이는 체감할 수 없는 수준입니다.

그럼에도 실무에서 `some`을 적극 권장하는 이유는 **'가독성'** 때문입니다. 
명령형인 `for` 문과 달리, 선언적인 `some()`을 사용하면 코드를 보자마자 **"아, 이 로직은 배열 안에서 조건을 확인하고 boolean을 반환하려는 거구나!"** 하고 동료 개발자가 작성자의 의도를 즉시 파악할 수 있어 유지보수성이 크게 향상됩니다.
