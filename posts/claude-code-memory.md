## Claude Code의 메모리 파일

Claude Code는 세션을 시작할 때 여러 위치의 **메모리 파일**을 자동으로 불러옵니다.
메모리 파일은 `CLAUDE.md`와 `CLAUDE.local.md` 두 종류이며, 저장 위치에 따라 다음 4가지 계층으로 나뉩니다.

1. 관리형 정책 (Managed Policy)
2. 프로젝트 메모리 (Project Memory)
3. 사용자 메모리 (User Memory)
4. 프로젝트 로컬 메모리 (Project Local Memory)

각 계층은 적용 범위와 공유 방식이 다릅니다. 하나씩 정리해 보겠습니다.

---

## 1) 관리형 정책 (Managed Policy)

조직 전체에 적용되는 지침을 담는 계층으로, 보통 IT나 데브옵스 팀에서 관리합니다.
운영체제별 경로는 다음과 같습니다.

| OS | 경로 |
| --- | --- |
| macOS | `/Library/Application Support/ClaudeCode/CLAUDE.md` |
| Linux | `/etc/claude-code/CLAUDE.md` |
| Windows | `C:\Program Files\ClaudeCode\CLAUDE.md` |

조직의 모든 사용자가 이 설정을 공유하므로, 보안 정책이나 코드 표준 같은 **전사적 규칙**을 정의하는 데 활용합니다.

> 대규모 모노레포에서 불필요한 하위 디렉터리의 `CLAUDE.md` 로딩을 막으려면
> `settings.json`의 `claudeMdExcludes`로 특정 경로를 제외할 수 있습니다.
> 단, **관리형 정책은 이 설정으로 제외할 수 없으며 항상 적용**됩니다.

---

## 2) 프로젝트 메모리 (Project Memory)

프로젝트 루트의 `./CLAUDE.md` 또는 `./.claude/CLAUDE.md` 파일에 저장되며,
**소스 제어를 통해 팀에 공유**됩니다.

- 프로젝트별 코딩 스타일
- 아키텍처 결정
- 빌드 명령어

이런 내용을 문서화하기에 적합합니다. 깃 저장소에 커밋되므로 팀원 모두가 동일한 환경에서 작업할 수 있습니다.

하위 디렉터리에도 `CLAUDE.md`를 둘 수 있는데, 이 파일은 세션 시작 시가 아니라
**해당 디렉터리의 파일을 읽을 때 온디맨드(on-demand)로 로딩**됩니다.

---

## 3) 사용자 메모리 (User Memory)

`~/.claude/CLAUDE.md` 파일에 저장되며, 개인의 **모든 프로젝트에 적용되는 기본 설정**을 담습니다.
개발자 본인만 사용하는 영역으로 다음과 같은 내용을 정의합니다.

- 선호하는 에디터 스타일
- 자주 사용하는 단축키 패턴
- 개인적인 워크플로

---

## 4) 프로젝트 로컬 메모리 (Project Local Memory)

`./CLAUDE.local.md` 파일에 저장되며, **현재 프로젝트에 대한 개인적인 기본 설정**을 담습니다.
이 파일은 깃에 커밋되지 않으므로 다음과 같은 개인 설정에 활용합니다.

- 개인 샌드박스 URL
- 선호하는 테스트 데이터
- 로컬 개발 환경 설정

---

## 로딩과 우선순위

메모리 파일은 Claude Code 시작 시 자동으로 로드됩니다.
충돌이 발생하면 **더 구체적인(좁은 범위) 설정이 더 높은 우선순위**를 가집니다.

```
프로젝트 로컬 > 프로젝트 > 사용자 > 관리형 정책
```

즉,

- 프로젝트 로컬 설정이 프로젝트 설정보다 우선하고
- 프로젝트 설정이 사용자 설정보다 우선하며
- 사용자 설정이 조직의 관리형 정책보다 우선합니다.

좁은 범위일수록 그 상황에 더 맞는 설정이라는 전제이기 때문에, 가장 구체적인 설정을 우선 적용하는 것입니다.

---

## 정리

| 계층 | 파일 | 공유 범위 | 깃 커밋 |
| --- | --- | --- | --- |
| 관리형 정책 | `CLAUDE.md` (OS별 경로) | 조직 전체 | — |
| 프로젝트 메모리 | `./CLAUDE.md`, `./.claude/CLAUDE.md` | 팀 | O |
| 사용자 메모리 | `~/.claude/CLAUDE.md` | 개인(모든 프로젝트) | — |
| 프로젝트 로컬 메모리 | `./CLAUDE.local.md` | 개인(현재 프로젝트) | X |

메모리 계층을 이해하면, 어떤 설정을 어디에 둘지 명확해집니다.
**팀과 공유할 규칙은 프로젝트 메모리에, 나만의 설정은 사용자/프로젝트 로컬 메모리에** 두는 것이 핵심입니다.
